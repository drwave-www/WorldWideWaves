actual fun createPlatformNotificationManager(): NotificationManager

iOS implementation of createPlatformNotificationManager.

iOS Safety CRITICAL

DOES NOT USE KOIN DI - Creates instance directly to prevent iOS deadlocks:

  • ❌ WRONG: return get<NotificationManager>() - Could cause deadlock

  • ✅ CORRECT: return IOSNotificationManager() - Direct instantiation

Why Direct Instantiation?

UNUserNotificationCenter is a platform singleton with no dependencies:

  • No constructor parameters needed

  • No DI required

  • Safe to create directly

Pattern Consistency

Follows the same pattern as SystemClock:

  • Class-based implementation (not object)

  • Lazy initialization of platform APIs

  • No DI in critical paths
